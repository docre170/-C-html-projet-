Module 3 : Codage et sécurité des données

C’est quoi la sécurité des données?

Quand on envoie ou garde des informations (comme un mot de passe, un message ou un numéro de carte), il faut les protéger pour que personne d’autre ne les lise ou les vole.
Exemple : C’est comme écrire un secret en code pour que seul ton ami puisse le comprendre.

différence entre codage et cryptage

  • - Codage = transformer une donnée pour qu’elle soit plus facile à transporter ou stocker.
  • - Cryptage (ou chiffrement) = transformer une donnée pour la cacher.

2. Chiffrement (cryptage)

C’est une technique pour cacher des données en les mélangeant.
Exemple simple :

  • - Message original : CHAT
  • - Code secret : décaler chaque lettre de 3 : C → F, H → K, A → D, T → W.
    - Résultat : FKDW
  • - Seule la personne qui connaît la clé (le nombre 3 peut retrouver le message.

3. Quelques méthodes de chiffrement :

a. Chiffrement symétrique :

- Même clé pour coder et décoder.
- Exemple : Clé = 1234 → coder et lire avec la même.

b. *Chiffrement asymétrique* :

Deux clés différentes : clé publique (pour coder) et clé privée (pour décoder).
- Exemple : Tu laisses ta clé publique sur Internet, mais seule ta clé privée peut ouvrir le message.

4. Autres moyens de protection :

- Hachage : transforme un mot (ex: mot de passe) en une longue suite de lettres/chiffres (ex: `5F4DCC...`), irréversible.
- Mot de passe fort : mélange lettres, chiffres, symboles.
- Pare-feu, antivirus : protègent ton ordinateur contre les voleurs de données.

5. Pourquoi coder les données ?

1. Pour empêcher les pirates de lire les messages.
2. Pour vérifier qu’un message n’a pas été modifié.
3. Pour prouver qui a envoyé le message (authenticité).
