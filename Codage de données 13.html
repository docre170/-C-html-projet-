Module 1 : Introduction au codage des données

Imagine que l’ordinateur ne comprend que deux choses : des 0 et des 1. C’est un peu comme une langue secrète. Le codage des données, c’est donc transformer ce que nous comprenons (texte, images, sons, vidéos…) en une forme que l’ordinateur peut lire et utiliser : une suite de 0 et 1, qu’on appelle le binaire.

Pourquoi coder les données ?

  1. Pour communiquer avec l’ordinateur.
  2. Pour stocker des informations (comme un fichier texte ou une image).
  3. Pour transmettre des données entre ordinateurs (comme sur Internet).
  4. Pour protéger ou compresser les informations (moins d’espace, plus de sécurité).

Exemple simple :
Si tu veux dire à l’ordinateur « A », il ne comprend pas la lettre, mais il comprend :
- A = 65 en code ASCII (un ancien code standard)
- En binaire : 01000001
Donc quand tu tapes A, l’ordinateur voit 01000001. C’est ça le codage.

Différents types de données à coder :

- Texte (comme des lettres ou mots)
- Chiffres (nombres, calculs)
- Images (chaque pixel a une couleur codée)
- Sons (ondes transformées en nombres)
- Vidéos (images + sons codés)

Les bases du codage :

- Systèmes de numération : binaire, décimal, hexadécimal…
- Unités de données : bit (0 ou 1), octet (8 bits), kilo-octet, méga-octet…
- Codage standardisé : comme ASCII, UTF-8, pour les lettres
